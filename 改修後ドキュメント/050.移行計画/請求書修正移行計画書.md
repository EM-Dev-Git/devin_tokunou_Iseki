# 請求書修正移行計画書

## 1. 概要

本ドキュメントは、請求書修正機能の移行計画を定義したものです。熊本生産Ｇにおける発送運賃請求内容入力漏れの会計処理ミスの再発防止対応として実装する請求書の改善機能について、本番環境への移行計画を記載します。

## 2. 移行対象

### 2.1 データベース
- M_SEIKYUテーブルにTANI列（VARCHAR(10)）を追加

### 2.2 アプリケーション
- 請求マスター登録画面（RegisterSeikyuMasterForm.vb）
- 請求書出力（OutputSeikyuReportS.rdlc、OutputSeikyuReportU.rdlc）
- データグリッドビュー（UcDataGridView.vb）

## 4. 移行手順

### 4.1 データベース移行

#### 4.1.1 バックアップ取得

```sql
-- バックアップテーブル作成
SELECT * INTO M_SEIKYU_BACKUP FROM M_SEIKYU;

-- バックアップ確認
SELECT COUNT(*) FROM M_SEIKYU;
SELECT COUNT(*) FROM M_SEIKYU_BACKUP;
```

#### 4.1.2 テーブル変更

```sql
-- TANI列追加
ALTER TABLE M_SEIKYU ADD TANI VARCHAR(10) NOT NULL DEFAULT '個';

-- 既存データの単位を「個」に設定
UPDATE M_SEIKYU SET TANI = '個';

-- デフォルト制約の削除
ALTER TABLE M_SEIKYU ALTER COLUMN TANI DROP DEFAULT;

-- 変更確認
SELECT TOP 10 * FROM M_SEIKYU;
```

### 4.2 アプリケーション移行

#### 4.2.1 配布前準備

1. アプリケーションのビルド
   - リリースビルドの作成
   - バージョン番号の更新

2. 配布パッケージの作成
   - 必要なファイルの確認
   - インストーラーの作成（必要な場合）

#### 4.2.2 配布手順

1. 既存アプリケーションのバックアップ
   - 実行ファイル、設定ファイルのバックアップ

2. 新バージョンの配布
   - 各クライアントへの配布方法（手動/自動）
   - 配布スケジュール（一斉/段階的）

3. 初回起動時の注意点
   - 設定ファイルの移行
   - 初期設定の確認

## 5. 移行リスクと対策

### 5.1 データベース関連リスク

| リスク | 影響度 | 発生確率 | 対策 |
|--------|--------|----------|------|
| テーブル変更失敗 | 高 | 低 | バックアップからの復旧手順を準備 |
| データ不整合 | 中 | 低 | 移行後のデータ検証クエリを準備 |
| パフォーマンス低下 | 低 | 低 | 事前に性能テストを実施 |

### 5.2 アプリケーション関連リスク

| リスク | 影響度 | 発生確率 | 対策 |
|--------|--------|----------|------|
| 起動エラー | 高 | 低 | 旧バージョンへの切り戻し手順を準備 |
| 機能不具合 | 中 | 中 | テスト環境での十分な検証 |
| ユーザー混乱 | 低 | 中 | 事前の説明会とマニュアル配布 |

## 6. 切り戻し計画

### 6.1 切り戻し判断基準

以下の状況が発生した場合、切り戻しを検討する：

1. データベース変更後にシステムが正常に動作しない
2. 新機能に重大な不具合が発見された
3. 業務に支障をきたす性能低下が発生

### 6.2 データベース切り戻し手順

```sql
-- 既存テーブルの退避
SELECT * INTO M_SEIKYU_TEMP FROM M_SEIKYU;

-- バックアップからの復元
DROP TABLE M_SEIKYU;
SELECT * INTO M_SEIKYU FROM M_SEIKYU_BACKUP;

-- 復元確認
SELECT COUNT(*) FROM M_SEIKYU;
```

### 6.3 アプリケーション切り戻し手順

1. 旧バージョンの再配布
   - バックアップからの復元
   - 配布方法（手動/自動）

2. 切り戻し後の確認
   - 基本機能の動作確認
   - データ整合性の確認

## 7. 移行後の確認項目

### 7.1 データベース確認

- TANI列が正しく追加されていることの確認
- 既存データの単位が「個」に設定されていることの確認
- インデックスやパフォーマンスの確認

### 7.2 アプリケーション確認

- 請求マスター登録画面での単位入力確認
- 数量×単価の自動計算機能確認
- 金額項目の読み取り専用確認
- 請求書出力での単位表示確認

### 7.3 業務フロー確認

- 請求マスター登録から請求書出力までの一連フロー確認
- 異なる単位での出力確認
- エラー処理の確認
