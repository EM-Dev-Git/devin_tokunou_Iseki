# コード改修箇所

本ドキュメントは、請求書フォーマットの修正における具体的なコード改修箇所を記載しています。

## 1. OutputSeikyuReportU.rdlc

### 修正前
```xml
<TextRun>
  <Value>=Format(Fields!KOSU.Value,"#,##0")</Value>
  <Style>
    <FontFamily>ＭＳ ゴシック</FontFamily>
    <FontSize>9pt</FontSize>
    <Color>=IIf(Fields!KOSU.Value &lt; 0,"Red","Black")</Color>
  </Style>
</TextRun>
```

### 修正後
```xml
<TextRun>
  <Value>=Format(Fields!KOSU.Value,"#,##0") &amp; " | " &amp; Fields!TANI.Value</Value>
  <Style>
    <FontFamily>ＭＳ ゴシック</FontFamily>
    <FontSize>9pt</FontSize>
    <Color>=IIf(Fields!KOSU.Value &lt; 0,"Red","Black")</Color>
  </Style>
</TextRun>
```

### 修正内容の説明
- 数量（KOSU）と単位（TANI）を同じ列に表示するように変更
- 数量と単位の間に「 | 」（縦線）を挿入して区切り
- 既存のフォーマット設定（#,##0）と色設定（負の値は赤色表示）は維持

## 2. 関連するコード

### clsDirectPrintSeikyu.vb
このファイルでは修正は不要でした。既に以下のSQLクエリで単位（TANI）フィールドを取得しています：

```vb
Dim text2 As String = "SELECT H.TORI_CD, H.SEIKYU_YYYYMM, H.SEIKYU_TYPE, H.SEIKYU_NO, H.KAZEI, H.HIKAZEI, H.SYOHIZEI, H.GOUKEI, M.SEIKYU_SQNO, M.SAKU_KBN, M.UCHIWAKE, M.SURYO, M.TANI, "
text2 += " M.TANKA, M.KINGAKU, M.MEISAI_UMU, M.SAKI_CD, M.TEKIYO, T.TORI_NAME"
```

### RegisterSeikyuMasterForm.vb
このファイルでは修正は不要でした。既に以下の単位（TANI）フィールドの入力チェックが実装されています：

```vb
' 単位チェック
If IsDBNull(row.Cells("TANI").Value) OrElse String.IsNullOrEmpty(Conversions.ToString(row.Cells("TANI").Value)) Then
    Me.lblMessage.Text = "単位を入力してください。"
    Me.UcDgv.CustDgv.CurrentCell = row.Cells("TANI")
    Return False
End If
```

## 3. 修正の影響範囲

### 影響を受けるファイル
- OutputSeikyuReportU.rdlc

### 影響を受けない（既に対応済み）ファイル
- clsDirectPrintSeikyu.vb（既にTANIフィールドを取得）
- RegisterSeikyuMasterForm.vb（既にTANIフィールドの入力チェックを実装）
- OutputSeikyuReportS.rdlc（既に同様の形式で表示）

## 4. テスト方法

1. 請求マスター登録画面で単位を入力して請求データを登録
2. 請求書出力機能で請求書を出力
3. 出力された請求書の数量列に「999 | 個」のような形式で表示されることを確認

## 5. 期待される結果

修正後の請求書フォーマットでは、数量列に「999 | 個」のような形式で数量と単位が表示されます。これにより、数量の解釈ミスを防止し、会計処理ミスの再発を防止します。
