# 生産受払システム機能追加 改修要件一覧

## 1. 背景・目的
熊本生産Ｇにおける発送運賃請求内容入力漏れの会計処理ミスの再発防止対応として、請求マスタ機能の改善を行う。

## 2. 発生要因
1. 担当者による請求金額入力漏れ
2. 請求マスタにおいて数量×単価入力後の自動計算機能がない（発生真因）

## 3. 対応内容（機能追加）
1. 単価、数量入力後、金額が自動計算される機能を追加する
2. 単価、数量いずれか未入力の場合、入力を促す背景反転色付けする機能を追加する
3. 金額は手入力ができないようグレーアウトにする
4. 請求書記載の数量の単位については、「単位」単独の項目を追加する
5. 追加項目の単位は入力必須項目とし、未入力の場合背景色付けエラーにする

## 4. 修正対象ドキュメント
以下のドキュメントに対して修正が必要です：

1. **請求処理フロー**
   - ファイル: `/ドキュメント/010.システム全般/請求処理フロー.xlsx`
   - 修正内容: 請求マスタ登録時の処理フローに自動計算機能を追加

2. **請求マスター登録画面定義書**
   - ファイル: `/ドキュメント/020.画面設計書/受払システム_画面定義書(17.請求マスター登録).xlsx`
   - 修正内容:
     - 画面レイアウトに「単位」項目の追加
     - 数量×単価の自動計算機能の追加
     - 金額項目のグレーアウト設定
     - 必須入力項目（単価、数量、単位）の背景色設定

3. **請求書出力画面定義書**
   - ファイル: `/ドキュメント/020.画面設計書/受払システム_画面定義書(62.請求書出力).xlsx`
   - 修正内容: 出力される請求書に「単位」項目を追加

4. **DB論理設計書**
   - ファイル: `/ドキュメント/030.DB論理設計書/受払管理システム_DB論理設計書.xlsx`
   - 修正内容: 請求マスタテーブルに「単位」項目を追加
